### 04.10 OpenFeign 의존성을 추가하는 데 안 됩니다..
> 일단 전에 Settings.Gradle을 Root로 만들어 두고 각 모듈에서 build.gradle을 관리합니다.

저는 현재 Server, user(예정), product, order 이렇게 4개의 모듈이 있습니다.
이 때 server가 core과 같은 의미가 될 예정입니다.

근데 저는 사실 gradle을 지금은! 따로 관리하고 싶어서 Root 에 Build.gradle을 관리 안 하고 싶은데.. 
근데 생각해보니 일반적으로 상위 -> 하위 이렇게 모듈이 관리 될 텐데..?
근데 또 Server 모듈이 Core라고 정의 했는데 `implementation project(:server)`을 하는게 맞나?? 라는 생각도 같이.. ㅠ

#### Answer
일단 Server는 Core 모듈이 아닙니다.
Server은 Eureka 서버를 관리하는 "인프라 서비스"입니다.
Core 모듈은 공통 처리 또는 예외나 공통 코드를 다루기 때문에 common이라고도 부릅니다.

그렇기 때문에 하위 모듈이 공통 모듈을 참조하기에 `projdct(:core)`을 쓰는거지 다른 도메인 영역이 Erureka 서버나 인프라 서버를 참조할 이유가 없습니다.

그냥 Root에 **비어있는 Build.gradle**만 넣어줘도 잘 인식이 됩니다!
즉 지금의 문제는 내용은 하위 모듈에서 각자 관리했지만, IDE가 하위 모듈들을 다 알게 디ㅗㅂ니다!!
